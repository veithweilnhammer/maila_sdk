MAILA Sandbox

Human-computer interactions encode latent mental states. This sandbox lets you run MAILA end to end: collect human-computer interactions, upload them, and generate model predictions.

About MAILA:
  • MAILA infers latent mental states from everyday human-computer interactions.
  • This sandbox is for testing ingestion and inference workflows described in our preprint.
  • Outputs are model estimates for research/product experimentation, not a clinical diagnosis.

Actions:
  • Start recording: starts capture and uploads event batches while you move the cursor.
  • Stop: stops capture and ends active segmentation.
  • Predict (flush + /predict): predicts from the currently recorded user_id/session_id.
  • New session: keeps the user, rotates session_id, and starts a fresh trial window.
  • Predict from uploaded file: runs inference directly from a local recording file via /predict-upload.
Pixelblot recorder (what this page embeds):
  • This page includes pixelblot-recorder.js and creates a PixelblotRecorder instance on Start recording.
  • While running, it samples pointermove (mouse/touch/stylus) and uploads event batches to /exp_main every ~flushMs (default: 1s).
  • Events are grouped into fixed windows (default: segmentMs = 10s) via segment_start/segment_end.
  • Predict flushes pending events, then calls /predict for the current user_id/session_id and selected model_group.

Upload file requirements:
  • Preferred format: .jsonl (JSON Lines, one JSON event per line).
  • Also accepted by /predict-upload: .json, .txt.
